ВСЕ! что связанно с ui храниться здесь.

Больше нигде в приложении не должна быть реализованна работа c версткой и виджетами.

Работа с экранами/виджетами должна быть через Stateless Widget по возможности. Внутри BlocBuilder со специальным Bloc для этого экрана/виджета, и в нем уже прописываем логику поведения экрана/виджета. Этот локальный блок провайдится через BlocProvider.

Если нужно сделать какую то утилитку связанную с ui, создаем ее в папке ui/utils. В корневой папке utils не должно быть функционала связанного с ui.

Такая логика позволить четко разграничить работу с бизнес логикой и версткой.
